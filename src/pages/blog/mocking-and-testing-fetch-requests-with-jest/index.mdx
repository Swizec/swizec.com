First, a little background



## Why even test requests?

Integration testing. Also known as the only useful testing. üòõ



Semantics aside, most bugs happen on the edges. The touch points between systems. Interfaces between modules.



What bigger in-between point than calling an API? Lots can go wrong. Server can fail, network down, strange response codes, or just corruption on the wire.



Take for example this `isAuthenticated` method from a library I built at work.

Is the user authenticated? Check local cookies. That&#x27;s always gonna work.



And if cookies _look_ correct, you gotta ask the server if they&#x27;re _actually_ correct. User might have been banned, deleted, access revoked, or just changed their password.



That API request is the most likely failure point.



Wanna _really_ test this method? Gonna have to test the fetch.



## Why mock fetch requests?



A pure unit testing approach says you should never mock requests to other systems.



You assume the request works and test that your function makes the request. Then you test the function that&#x27;s called after the fetch.



And you enter the unit testing fallacy.



![](./img/media-Q0Mpd9MWuERyg-giphy.gif)



Besides, structuring your code to allow a pure unit testing approach makes it harder to read, understand, and tedious to work with. You&#x27;d need something like this:

Keep your code simple and mock the request instead. Hook into the `fetch()` API and manipulate what it returns. That is the way to testing bliss my friend.



## How to mock fetch requests in Jest



After more hours of trial and error than I dare admit, here&#x27;s what I found: You&#x27;ll need to mock the `fetch` method itself, write a mock for each request, and use a fetch mocking library.



### install



The library that worked best for me was [`fetch-mock`](https://github.com/wheresrhys/fetch-mock).



[![Click through for source](./img/techletter.app-screenshot-1582908305188.png)](https://carbon.now.sh/?bg=rgba(255,255,255,1)&amp;t=seti&amp;l=shell&amp;ds=true&amp;wc=true&amp;wa=true&amp;pv=48px&amp;ph=32px&amp;ln=false&amp;code=yarn%20add%20--dev%20fetch-mock)



### mock



You tell Jest to use a mock library like this:



[![Click through for source](./img/techletter.app-screenshot-1582908305230.png)](https://carbon.now.sh/?bg=rgba(255,255,255,1)&amp;t=seti&amp;l=typescript&amp;ds=true&amp;wc=true&amp;wa=true&amp;pv=48px&amp;ph=32px&amp;ln=false&amp;code=%2F%2F%20src%2F__mocks__%2Fisomorphic-fetch.js%0A%0Aconst%20fetchMock%20%3D%20require(%22fetch-mock%22)%3B%0A%0Aconst%20fetch%20%3D%20fetchMock.sandbox()%3B%0Amodule.exports%20%3D%20fetch%3B)



Jest \\imports this file instead of `isomorphic-fetch` when running your code. Same approach works to replace any other library.



Put a file of &amp;lt;library name&gt; in `src/__mocks__` and that file becomes said library. In this case we&#x27;re replacing the `isomorphic-fetch` library with a `fetch-mock` sandbox.



So when your code says



[![Click through for source](./img/techletter.app-screenshot-1582908305371.png)](https://carbon.now.sh/?bg=rgba(255,255,255,1)&amp;t=seti&amp;l=typescript&amp;ds=true&amp;wc=true&amp;wa=true&amp;pv=48px&amp;ph=32px&amp;ln=false&amp;code=\import%20fetch%20from%20&#x27;isomorphic-fetch&#x27;)



It&#x27;s actually getting the sandbox ü§ò



You&#x27;ll need to \\import _a_ fetch to support mocking ‚Äì can&#x27;t rely on the global `window.fetch` I&#x27;m afraid. You can&#x27;t use that in test environments anyway since it doesn&#x27;t exist.



### types



Using TypeScript in tests, you&#x27;ll have to jump through another hoop: TypeScript doesn&#x27;t understand that isomorphic-fetch is mocked and gets confused.



You have to tell the type system that _&quot;Hey, this is actually fetch-mock. All is well.&quot;_



[![Click through for source](./img/techletter.app-screenshot-1582908305261.png)](https://carbon.now.sh/?bg=rgba(255,255,255,1)&amp;t=seti&amp;l=null&amp;ds=true&amp;wc=true&amp;wa=true&amp;pv=48px&amp;ph=32px&amp;ln=false&amp;code=%2F%2F%20src%2F__tests__%2FAuthHelper.spec.ts%0A%0A\import%20_fetchMock%20from%20&#x27;isomorphic-fetch&#x27;%20%2F%2F%20\imports%20the%20sandbox%0Atype%20FetchMock%20%3D%20\import(&#x27;fetch-mock&#x27;)%20%2F%2F%20\imports%20type%20info%0A%0Aconst%20fetchMock%20%3D%20_fetchMock%20as%20FetchMock%20%2F%2F%20casts%20as%20correct%20type)



Convoluted but how else is TypeScript supposed to know `isomorphic-fetch` is actually `fetch-mock` ...



PS: I&#x27;m assuming Jest because it&#x27;s become the industry standard for JavaScript testing in the past few years.
